# Roadmap

## Research Direction

This roadmap is a working research backlog, not a fixed curriculum. Tracks can change as investigations produce better questions.

| Area | Focus | Possible Outcome |
| --- | --- | --- |
| Agent loops | Observe-plan-act-evaluate cycles | Loop models and diagrams. |
| Context engineering | Prompt, file, task, and summary context | Context strategy notes. |
| Memory and state | Session, project, and long-term memory patterns | Memory design sketches. |
| Tool use and execution | Tool schemas, validation, side effects, and errors | Tool-use principles. |
| Runtime architecture | Process, sandbox, workspace, and orchestration models | Architecture review criteria. |
| Pi deep dive | Pi-style agent runtime design | System map and runtime traces. |
| Coding-agent UX | Review, approval, interruption, and collaboration flows | UX control patterns. |
| Multi-agent systems | Delegation, parallelism, and coordination | Coordination tradeoff notes. |
| Evaluation | Task suites, traces, rubrics, and regression checks | Evaluation plan. |
| Security and trust | Permissions, secrets, sandboxing, and auditability | Trust boundary review. |

## Candidate Milestones

- First architecture comparison notes.
- Pi deep-dive trace set.
- Evaluation plan draft.
- Architecture synthesis and project retrospective.
